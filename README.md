# stockbot-long-backend

台股 ETF (006208, 00830) 長線 + 波段交易 backend。
排程拉 TWSE 歷史價量、跑 baseline 加減碼策略、提供 REST API + Prometheus metrics 給前端與監控使用。
打包成 docker compose (MariaDB + Go app + Caddy 自動 HTTPS)，本機 / 正式機共用同一份 yaml。

## 快速啟動

```bash
cp .env.example .env
docker compose up -d --build
```

預設 `.env.example` 是「本機 HTTP only」設定，可直接跑。第一次啟動會等 5–10 分鐘回補 TWSE 歷史資料，
完成後 `curl http://localhost:8080/health` 應回 `{"status":"ok"}`。

完整步驟、正式機 HTTPS、維運指令請見 [docs/deployment.md](docs/deployment.md)。

## 目前的交易演算法

這個系統監控兩檔台股大盤型 ETF（006208、00830），核心邏輯是「**先判斷市場處於上漲趨勢還是下跌趨勢，再採用不同的進出場打法**」。兩檔標的各自獨立判斷，每個交易日以**當日收盤價**計算一次。

**問題設定（每月解鎖新資金）**：期初投入 100,000 元，之後在**每個月的第一個交易日**再解鎖 2,500 元可動用資金（模擬定期定額注資）。因為有持續的外部注資，回測報酬一律用**資金加權年化報酬（MWR / XIRR）**、回撤用 **NAV 單位淨值**（扣除注資灌高的真實投資回撤）；對照組 B&H 為「資金一解鎖就立刻等權買滿」。詳見 [docs/backtest.md](docs/backtest.md)。

需要用到的指標只有四個：**10 日均線**、**200 日均線**、**近 252 個交易日的最高收盤價**（約一年），以及**每檔持有期間出現過的最高收盤價**。

### 第一步：判斷市場趨勢（多頭或空頭）

以當日收盤價與 **200 日均線**的相對位置判斷市場狀態：
- 收盤價 **> 200 日均線** → 視為**多頭**（上漲趨勢）
- 收盤價 **≤ 200 日均線**，或資料不足 200 天 → 視為**空頭**（下跌趨勢）

單日穿越即立刻翻轉，不設緩衝區。後續所有買賣決策，都依據這個趨勢狀態切換。

### 第二步：判斷是否買進

**須同時滿足下列三項才會買進：**
1. 當日有有效收盤價，且已能算出 10 日均線。
2. 價格門檻：多頭時收盤價需 **< 10 日均線 × 1.05**；空頭時需 **< 10 日均線**（門檻較嚴格）。
3. 冷卻期：距離上次買進**同一檔**至少間隔 **14 個交易日**（多頭、空頭皆同）。**但每檔在「近 365 個日曆日」內另給 2 次「打破冷卻」額度**，可在冷卻內提前買進、撿回被錯過的深跌買點。

**買進金額（多頭、空頭都買「目前現金的一定比例」，不是固定金額）：**

- **多頭：買「目前現金的 25%」。**
- **空頭：買「目前現金的 2% × 跌幅深度權重」**，跌越深、投入的現金比例越大：

  | 距一年高點的跌幅 | 深度權重 | 投入（佔目前現金） |
  | --- | --- | --- |
  | 0–10% | ×1 | 2% |
  | 10–20% | ×2.5 | 5% |
  | 20–30% | ×6.25 | 12.5% |
  | 30–40% | ×15.6 | 31% |
  | 超過 40% | ×39 | 78% |

  （深度權重為倍率 **2.5** 的幾何曲線。）

**換算實際下單：**
- **換算股數** = 四捨五入（買進金額 ÷ 當日收盤價）；若現金不足，改買負擔得起的最大股數（永遠不借錢）。

### 第三步：判斷是否賣出（買進後以更新後的持股判斷，依當日趨勢分兩種打法）

賣出規則**隨多頭／空頭而不同**，每種趨勢各只有一條規則：

**🟢 多頭趨勢：只做「+100% 獲利了結」**
- 觸發條件：相對**持倉最低成本獲利 ≥ 100%**（價格翻倍）。
- 賣出量：**當前持股的 33%**（分批出場）。
- 多頭**不設移動停利**。

**🔴 空頭趨勢：只做「8% 移動停利」**
- 啟動條件：該檔**曾經獲利達 20% 以上**才武裝（不停損剛逢低買進、尚未獲利的部位）。
- 觸發賣出：當日收盤價 **≤ 持有期間最高收盤價 ×（1 − 8%）** → **將該檔全數賣出**。

### 風險控管鐵律

僅使用帳戶現有現金，**不使用融資（不借錢）**；當計算出的目標股數超出可動用資金時，僅買進負擔得起的部分。

### 一句話總結

> **上漲趨勢買「目前現金的 25%」；下跌趨勢買「目前現金的 2% × 跌幅深度權重」（跌越深比例越大，最深達現金 78%）；冷卻 14 天，另給每年 2 次「打破冷卻」額度；多頭獲利 100% 分批賣 33%，下跌趨勢以 8% 移動停利保護獲利。**

**回測表現**（006208 + 00830，2019-05 ~ 2026-06，約 7 年，未計交易成本）：

| 指標 | Ours（策略） | B&H（資金一解鎖就買滿） |
| --- | --- | --- |
| 全期期末權益 | **$1,430,049** | $1,376,606 |
| 全期資金加權報酬（MWR） | **+33.2%** | +32.3% |
| 全期最大回撤（NAV） | **−22.1%** | −38.0% |
| 全期資金利用率 | 80.3% | 100% |
| walk-forward 中位 MWR | +25.1% | +26.1% |
| walk-forward 中位回撤 | **−22.5%** | −32.7% |
| walk-forward Calmar | **1.20** | 0.89 |
| Calmar 勝率 / 真擇時勝率 | 85.7% / 81.0% | — |

> walk-forward = 24 個月視窗、步進 3 月、共 21 個進場時點的中位數，用來檢驗「換任何進場時點都穩」。

策略以約 80% 的平均曝險，做到**期末權益與資金加權報酬都略勝「永遠滿倉」的 B&H**，而最大回撤只有 B&H 的約六成、Calmar 勝率 86%。把牛市買入比例由現金 10% 上調到 **25%** 後，資金利用率由 74% 升到 80%、閒置現金尾巴由 $791k 降到 $568k，在風險幾乎不變（NAV 回撤 −21.8%→−22.1%）下解決了「強多頭參與不足」。代價：最嚴格的回撤門檻（中位回撤 ≤60% B&H）放寬為實際 ~69%，是刻意以略高回撤換取更高資金利用率的取捨。

> 注意：以上為歷史資料回測結果，不代表未來績效；標的亦為事後選定的長期上漲 ETF。本內容僅供參考，不構成投資建議。
