# stockbot-long-backend

台股 ETF (00631L, 00830) 長線 + 波段交易 backend。
排程拉 TWSE 歷史價量、跑 baseline 加減碼策略、提供 REST API + Prometheus metrics 給前端與監控使用。
打包成 docker compose (MariaDB + Go app + Caddy 自動 HTTPS)，本機 / 正式機共用同一份 yaml。

## 快速啟動

```bash
cp .env.example .env
docker compose up -d --build
```

預設 `.env.example` 是「本機 HTTP only」設定，可直接跑。第一次啟動會等 5–10 分鐘回補 TWSE 歷史資料，
完成後 `curl http://localhost:8080/health` 應回 `{"status":"ok"}`。

完整步驟、正式機 HTTPS、維運指令請見 [docs/deployment.md](docs/deployment.md)。

## 目前的交易演算法

這個系統監控兩檔台股 ETF（00631L、00830），核心邏輯是「**先判斷市場處於上漲趨勢還是下跌趨勢，再採用不同的進出場打法**」。兩檔標的各自獨立判斷，每個交易日以**當日收盤價**計算一次。

**問題設定（每月解鎖新資金）**：期初投入 100,000 元，之後在**每個月的第一個交易日**再解鎖 2,500 元可動用資金。回測與上線都從**所有追蹤股票都已發行的那一天**起算。因為有持續的外部注資，績效這樣衡量：

- **資金加權年化報酬（MWR / XIRR）**：把每一筆現金流（期初投入、每月注入記為流出，期末清算記為流入）依發生日期折現，解出使淨現值為 0 的年化報酬率，即考慮「每筆錢實際投入多久」的真實年化報酬；注資為 0 時等同一般年化報酬（CAGR）。
- **NAV 單位淨值回撤**：把注資當成「以當日淨值申購新單位」（像基金申購），得到一條只反映投資績效、與注資時點無關的單位淨值曲線，再取最大回撤。回撤因此不會被持續注入的本金灌高，量到的是真實投資回撤。

對照組 B&H 為「資金一解鎖就立刻等權買滿」。詳見 [docs/backtest.md](docs/backtest.md)。

需要用到的指標只有四個：**10 日均線**、**200 日均線**、**近 252 個交易日的最高收盤價**（約一年），以及**每檔持有期間出現過的最高收盤價**。

### 第一步：判斷市場趨勢（多頭或空頭）

以當日收盤價與 **200 日均線**的相對位置判斷：
- 收盤價 **> 200 日均線** → **多頭**
- 收盤價 **≤ 200 日均線**，或資料不足 200 天 → **空頭**

### 第二步：判斷是否買進

**須同時滿足下列三項才會買進：**
1. 當日有有效收盤價，且已能算出 10 日均線。
2. 多頭時收盤價 **< 10 日均線 × 1.05**；空頭時 **< 10 日均線**。
3. 距離上次買進**同一檔**至少間隔 **14 個交易日**（多頭、空頭皆同）。但每檔在「近 365 個日曆日」內另給 **2 次**「打破冷卻」額度，可在冷卻內提前買進。

**買進金額（買「目前現金的一定比例」）：**
- 多頭：買「目前現金的 **25%**」。
- 空頭：買「目前現金的 **2% × 跌幅深度權重**」：

  | 距一年高點的跌幅 | 深度權重 | 投入（佔目前現金） |
  | --- | --- | --- |
  | 0–10% | ×1 | 2% |
  | 10–20% | ×2.5 | 5% |
  | 20–30% | ×6.25 | 12.5% |
  | 30–40% | ×15.6 | 31% |
  | 超過 40% | ×39 | 78% |

**換算股數** = 四捨五入（買進金額 ÷ 當日收盤價）。

### 第三步：判斷是否賣出（依當日趨勢分兩種打法）

**多頭趨勢：+100% 獲利了結**
- 觸發：相對**持倉最低成本獲利 ≥ 100%**。
- 賣出量：**當前持股的 33%**。
- 多頭不設移動停利。

**空頭趨勢：8% 移動停利**
- 啟動：該檔**曾經獲利達 20% 以上**才武裝。
- 觸發：當日收盤價 **≤ 持有期間最高收盤價 ×（1 − 8%）** → **將該檔全數賣出**。

### 一句話總結

> 上漲趨勢買「目前現金的 25%」；下跌趨勢買「目前現金的 2% × 跌幅深度權重」；冷卻 14 天、另給每年 2 次「打破冷卻」；多頭獲利 100% 分批賣 33%，空頭以 8% 移動停利出場。

**回測表現**（00631L + 00830，2019-05 ~ 2026-06，約 7 年，未計交易成本；00631L 為 2 倍槓桿 ETF）：

- **全期期末**：總可利用資金 **$312,500**（期初 10 萬 + 每月 2,500），最後結算收益 Ours **$1,687,836**、B&H $2,798,385。
- **walk-forward**：每個 24 個月視窗約 **$157,500** 可利用資金，21 個進場時點的中位資金加權報酬 Ours **+25.5%**、B&H +38.2%。

| 指標 | Ours（策略） | B&H（資金一解鎖就買滿） |
| --- | --- | --- |
| 全期最後結算收益 | $1,687,836 | **$2,798,385** |
| 全期資金加權報酬（MWR） | +36.9% | **+48.8%** |
| 全期最大回撤（NAV） | **−38.7%** | −51.4% |
| 全期資金利用率 | 80.5% | 100% |
| walk-forward 中位 MWR | +25.5% | **+38.2%** |
| walk-forward 中位回撤 | **−33.6%** | −43.3% |
| walk-forward Calmar | 0.75 | **1.08** |

> 全期 = 一條連續長區間；walk-forward = 24 個月視窗、步進 3 月、共 21 個進場時點的中位數，用來檢驗「換任何進場時點都穩」。

在 2 倍槓桿的 00631L 上，策略把最大回撤從 B&H 的 −51% 壓到 **−39%**（walk-forward −43%→**−34%**），但報酬與風險調整後（Calmar）**不及單純滿倉的槓桿 B&H**——強多頭中 2 倍槓桿本身極難超越。本策略參數是針對 1 倍大盤 ETF 調校，套到 2 倍槓桿標的偏保守。

> 注意：以上為歷史資料回測結果，不代表未來績效；標的亦為事後選定的標的。本內容僅供參考，不構成投資建議。
