# stockbot-long-backend

台股 ETF (006208, 00830) 長線 + 波段交易 backend。
排程拉 TWSE 歷史價量、跑 baseline 加減碼策略、提供 REST API + Prometheus metrics 給前端與監控使用。
打包成 docker compose (MariaDB + Go app + Caddy 自動 HTTPS)，本機 / 正式機共用同一份 yaml。

## 快速啟動

```bash
cp .env.example .env
docker compose up -d --build
```

預設 `.env.example` 是「本機 HTTP only」設定，可直接跑。第一次啟動會等 5–10 分鐘回補 TWSE 歷史資料，
完成後 `curl http://localhost:8080/health` 應回 `{"status":"ok"}`。

完整步驟、正式機 HTTPS、維運指令請見 [docs/deployment.md](docs/deployment.md)。

## 目前的交易演算法

這個系統監控兩檔台股大盤型 ETF（006208、00830），核心邏輯是「**先判斷市場處於上漲趨勢還是下跌趨勢，再採用不同的進出場打法**」。兩檔標的各自獨立判斷，每個交易日以**當日收盤價**計算一次。

**問題設定（每月解鎖新資金）**：期初投入 100,000 元，之後在**每個月的第一個交易日**再解鎖 2,500 元可動用資金（模擬定期定額注資）。因為有持續的外部注資，回測報酬一律用**資金加權年化報酬（MWR / XIRR）**、回撤用 **NAV 單位淨值**（扣除注資灌高的真實投資回撤）；對照組 B&H 為「資金一解鎖就立刻等權買滿」。詳見 [docs/backtest.md](docs/backtest.md)。

需要用到的指標只有四個：**10 日均線**、**200 日均線**、**近 252 個交易日的最高收盤價**（約一年），以及**每檔持有期間出現過的最高收盤價**。

### 第一步：判斷市場趨勢（多頭或空頭）

以當日收盤價與 **200 日均線**的相對位置判斷市場狀態：
- 收盤價 **> 200 日均線** → 視為**多頭**（上漲趨勢）
- 收盤價 **≤ 200 日均線**，或資料不足 200 天 → 視為**空頭**（下跌趨勢）

單日穿越即立刻翻轉，不設緩衝區。後續所有買賣決策，都依據這個趨勢狀態切換。

### 第二步：判斷是否買進

**須同時滿足下列三項才會買進：**
1. 當日有有效收盤價，且已能算出 10 日均線。
2. 價格門檻：多頭時收盤價需 **< 10 日均線 × 1.05**；空頭時需 **< 10 日均線**（門檻較嚴格）。
3. 冷卻期：距離上次買進**同一檔**至少間隔 **多頭 12 個交易日、空頭 14 個交易日**（多頭略短，讓每月解鎖的新資金更快部署）。**但每檔在「近 365 個日曆日（≈ 一年）」內另給 2 次「打破冷卻」額度**：仍在冷卻內若出現買進訊號，可動用一次額度提前買進，撿回被冷卻錯過的買點（滾動視窗計數，回測與上線語意一致）。

**買進的金額（多頭、空頭都買「目前現金的一定比例」，不是固定金額）：**

買「剩餘現金的某個百分比」在數學上永遠歸不了零（每次只花一部分），所以每個更深、更便宜的跌點**都還有錢買**，徹底解決「最該抄底的深跌時卻沒現金」的問題；同時讓買進隨現金減少自然減速，是把回撤壓低的主煞車。

- **多頭：買「目前現金的 25%」。**（注資情境下從 10% 上調，把每月解鎖的新資金更快投入、撐高資金利用率）
- **空頭：買「目前現金的 2% × 跌幅深度權重」**，跌越深、投入的現金比例越大：

  | 距一年高點的跌幅 | 深度權重 | 投入（佔目前現金） |
  | --- | --- | --- |
  | 0–10% | ×1 | 2% |
  | 10–20% | ×2.5 | 5% |
  | 20–30% | ×6.25 | 12.5% |
  | 30–40% | ×15.6 | 31% |
  | 超過 40% | ×39 | 78% |

  （深度權重為倍率 **2.5** 的幾何曲線；陡度 2.5 是空頭加碼的主要報酬引擎。）

**換算實際下單：**
- **換算股數** = 四捨五入（買進金額 ÷ 當日收盤價）；若現金不足，改買負擔得起的最大股數（永遠不借錢）。

### 第三步：判斷是否賣出（買進後以更新後的持股判斷，依當日趨勢分兩種打法）

賣出規則**隨多頭／空頭而不同**，每種趨勢各只有一條規則：

**🟢 多頭趨勢：只做「+100% 獲利了結」（讓贏家奔跑、賺多了才分批落袋）**
- 觸發條件：相對**持倉最低成本獲利 ≥ 100%**（價格翻倍）。
- 賣出量：**當前持股的 33%**（分批出場；若連續達標會逐日各賣 33%）。
- 多頭**不設移動停利**，讓獲利部位延續上漲。

**🔴 空頭趨勢：只做「8% 移動停利」（保護既有獲利、嚴控回撤）**
- 啟動條件：該檔**曾經獲利達 20% 以上**（持有期間最高收盤價 ÷ 持倉最低成本 − 1 ≥ 0.20）才武裝；避免把剛逢低買進、尚未獲利的部位停損出場。
- 觸發賣出：當日收盤價 **≤ 持有期間最高收盤價 ×（1 − 8%）** → **將該檔全數賣出**。
- 空頭**不做 +100% 獲利了結**：要相對最低成本翻倍，價格幾乎必已站上 200 日均線＝多頭；實測 7 年回測空頭從未觸發過此條。

### 風險控管鐵律

僅使用帳戶現有現金，**不使用融資（不借錢）**；當計算出的目標股數超出可動用資金時，僅買進負擔得起的部分。

### 一句話總結

> **上漲趨勢順勢買進「目前現金的 25%」；下跌趨勢買進「目前現金的 2% × 跌幅深度權重（跌越深比例越大，最深達現金 78%）」——因為買的是現金比例，永遠留有銀彈，深跌時也接得動；冷卻多頭 12 天 / 空頭 14 天，另給每年 2 次「打破冷卻」額度撿回深跌買點；多頭獲利達 100% 分批賣出 33%，並在下跌趨勢以 8% 移動停利保護既有獲利。**

回測表現（每月解鎖 2,500 問題設定；walk-forward 滾動驗證，24 個月視窗、共 21 個視窗，未計交易成本）：資金加權報酬 **+26.5%**、最大回撤（NAV）**−22.1%**、Calmar **1.22**、Calmar 勝率 **95.2%**、真擇時勝率 **85.7%**（同期「資金一解鎖就立刻買滿」的 B&H 為 +26.1% / −32.7% / 0.89）。全期連續回測中策略期末權益（$1,389,264）已**反超 B&H**（$1,376,606）、資金加權報酬 +32.5% 亦略勝 B&H +32.3%，而最大回撤僅 −21.9%（B&H −38.0%）。把牛市買入比例從現金 10% 上調到 25%、牛市冷卻 14→12 天後，資金利用率由 74% 升到 **80%**（walk-forward 86%），閒置現金尾巴由 $791k 降到 $461k——在風險幾乎不變下解決了「強多頭參與不足」的問題。代價是最嚴格的回撤門檻（≤60% B&H 回撤）放寬為實際的 ~68%，屬刻意以略高回撤換取更高資金利用率的取捨。

> 注意：以上為歷史資料回測結果，不代表未來績效；標的亦為事後選定的長期上漲 ETF。本內容僅供參考，不構成投資建議。
