# stockbot-long-backend

台股 ETF (006208, 00830) 長線 + 波段交易 backend。
排程拉 TWSE 歷史價量、跑 baseline 加減碼策略、提供 REST API + Prometheus metrics 給前端與監控使用。
打包成 docker compose (MariaDB + Go app + Caddy 自動 HTTPS)，本機 / 正式機共用同一份 yaml。

## 快速啟動

```bash
cp .env.example .env
docker compose up -d --build
```

預設 `.env.example` 是「本機 HTTP only」設定，可直接跑。第一次啟動會等 5–10 分鐘回補 TWSE 歷史資料，
完成後 `curl http://localhost:8080/health` 應回 `{"status":"ok"}`。

完整步驟、正式機 HTTPS、維運指令請見 [docs/deployment.md](docs/deployment.md)。

## 目前的交易演算法（白話版）

這個系統監控兩檔台股大盤型 ETF（006208、00830），核心邏輯是「**先判斷市場處於上漲趨勢還是下跌趨勢，再採用不同的進出場打法**」。兩檔標的各自獨立判斷，每天以收盤價計算一次。

### 第一步：判斷市場趨勢（多頭或空頭）

計算一條 **200 日均線**（過去約一年的平均收盤價，代表長期趨勢方向），並以當日收盤價與它的相對位置判斷市場狀態：
- 收盤價**站上** 200 日均線 → 視為**多頭**（上漲趨勢）
- 收盤價**跌破** 200 日均線 → 視為**空頭**（下跌趨勢）

後續所有買賣決策，都依據這個趨勢狀態切換。

### 多頭：順勢進場，積極佈局

- **進場條件**：當日股價未明顯高於 10 日均線（在 10 日均線的 1.05 倍以內）即可買進；同一檔標的至少間隔 14 天才會再次進場。
- **進場規模**：每次買進固定一筆，並**隨帳戶總值成長等比放大**（複利）。
- **設計理由**：上漲趨勢中後續續漲的機率較高，不宜長期持有現金空等回檔，應將閒置資金投入市場。

### 空頭：謹慎進場，分批逢低承接

- **進場條件**：股價需**確實跌破 10 日均線**才買進（門檻較多頭嚴格）。
- **進場規模**：依「**距過去一年最高點的跌幅**」決定，跌幅越大、單次買進越多（跌幅 0–10% 買進最少，跌幅超過 40% 買進最多）。
- **設計理由**：長線相對理想的進場點，多半出現在恐慌性下跌階段；價格越低，分批承接的力度越大。

### 賣出邏輯

1. **獲利倍增後分批了結**：當某檔標的自最低買進成本上漲達 **100%**，賣出當前持股的 **1/3**（之後若續漲會再分批賣出）。
2. **空頭時啟動移動停利**：在下跌趨勢中，對於**未實現獲利已超過 20%** 的持股，一旦自其波段高點回落 **8%**，即全數賣出，避免既有獲利回吐。多頭趨勢不啟用此條件，讓獲利部位延續上漲。

### 風險控管鐵律

僅使用帳戶現有現金，**不使用融資（不借錢）**；當計算出的目標股數超出可動用資金時，僅買進負擔得起的部分。

### 一句話總結

> **上漲趨勢順勢買進、並隨帳戶複利放大；下跌趨勢分批逢低承接；獲利達標分批了結，並在下跌趨勢以 8% 移動停利保護既有獲利。**

回測表現（過去約 7 年資料、walk-forward 滾動驗證）：年化報酬約 **+26%**、最大回撤約 **−23%**（同期「買進後持有不動」最大回撤約 −33%）。

> 注意：以上為歷史資料回測結果，不代表未來績效；標的亦為事後選定的長期上漲 ETF。本內容僅供參考，不構成投資建議。
