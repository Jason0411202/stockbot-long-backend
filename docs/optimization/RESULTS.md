# 自主參數掃描 — 結果與結論

> 對 Baseline 逢低加碼策略，把 [optimization-directions-100.md](optimization-directions-100.md) 的高優先方向
> 做成 **flag-gated 旋鈕**，用 walk-forward + benchmark 框架**逐一 sweep 調餐、勝者留、敗者丟棄**。
> 全程不依賴 MariaDB / docker：資料由 `cmd/fetch_data` 抓 TWSE 寫成本機 CSV 快取。

## 方法論

| 項目 | 設定 |
| --- | --- |
| 標的 | 006208、00830（CSV 快取，2018-12 ~ 2026-06；共同有效期約 2019-06 起） |
| 評估 | walk-forward 24 個月視窗 / 每 3 個月滾動 / 最少 200 交易日 → **21 個視窗** |
| 對照組 | lump-sum B&H、naive DCA、**同曝險 blend**（防「抱現金」假優勢） |
| 選股留汰 | 候選值套在當前 champion 上跑 walk-forward；**勝者併入 champion、敗者丟棄**（coordinate-ascent，多輪） |
| 反作弊護欄 | 採納前必須：① 中位 CAGR 不比 champion 差 >12%（不犧牲報酬）② 真擇時勝率不比 champion 低 >8pp（不靠抱現金假贏） |

三種目標函式各跑一輪（`-objective`）：
- **gates**：最大化通過幾道 G1~G5 關卡（專案自身對「好策略」的定義）。
- **calmar**：最大化中位 Calmar（風險調整後報酬）。
- **composite**：`Calmar × 報酬參與率`（同時要風險調整 + 相對 B&H 有競爭力）。

## Baseline（現行策略）scorecard

```
中位 CAGR   Strat +10.1% | B&H +24.7% | 參與率 0.29
中位 MaxDD  Strat -12.9% | B&H -32.8%
中位 Calmar Strat 1.01  | B&H 0.94
中位 XIRR   Strat +33.4% | B&H +24.7%
平均曝險    35.5%
Calmar 勝率 57.1% | 真擇時勝率 61.9%
Gates: G1=✗ G2=✓ G3=✗ G4=✓ G5=✓  → 通過 3/5
```

**解讀**：現行策略風險低（回撤只有 B&H 的 ~40%）、且有**真實擇時能力**（贏同曝險 blend 62%），
但**嚴重投資不足**（只吃到 B&H 報酬的 29%、平均僅 35% 曝險）→ 卡在 G1（報酬參與）與 G3（Calmar 勝率）。

## 三目標的最終 champion

| 目標 | 採納的旋鈕 | 中位 CAGR | 中位 MaxDD | Calmar | Calmar勝率 | 真擇時 | 結論 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| **gates** | （無）| +10.1% | -12.9% | 1.01 | 57% | 62% | baseline 已是局部最佳 |
| **calmar** ⭐ | `sell_rsi_min=75` | +10.3% | -13.5% | **1.08** | **62%** | 62% | 風險調整變好、報酬不變 |
| **composite** | `ma_window=5` + `multiplier=3` + `sell_rsi_min=70` | **+17.2%** | -21.2% | 1.01 | 57% | 52% | 用風險換報酬 |

### 結論一：gates 目標 — 沒有任何單一旋鈕能讓 baseline 多過一道關卡
G1（要更多報酬）與 G3/G4（要風險調整 / 真擇時）天生互斥；掃描範圍內，baseline 對「5 道關卡」已是局部最佳。
這不是壞消息——它說明**現行策略並非隨手亂設，而是已落在自身評分標準的合理高原上**。

### 結論二：calmar 目標 — 唯一穩健的免費午餐是「賣在 RSI 強勢」⭐（已採納）
`sell_rsi_min=75`：獲利達 +100% 後，**只在 RSI(14) ≥ 75（超買）時才出場**（賣在強勢、讓贏家多跑一點）。
- 中位 Calmar **1.01 → 1.08**、Calmar 勝率 **57% → 62%**，**報酬與回撤幾乎不變**。
- **高原而非僥倖**：sweep 中 70 與 75 都改善且單調（50~65 完全無作用，因 +100% 出場點本就少落在 RSI<65）。
- 與 composite 目標**獨立重現**（composite 也採 70）→ 跨目標一致 = **低過擬合**。
- **已寫進 `config.yaml`**（`sell_rsi_period: 14` / `sell_rsi_min: 75`），上線決策同步生效。

### 結論三：composite 目標 — 要更有競爭力就得吃下更多風險
把進場均線縮短（`ma_window=5`，逢低訊號更密）並放大金額（`multiplier=3`）→ 曝險 35%→63%、
CAGR 10%→17%（參與率 0.29→0.66），但**最大回撤 13%→21%、真擇時 62%→52%、報酬離散 7%→13%**。
這是一條合法但**偏離「穩定、低風險」**的路線；若你願意承受更大回撤換取貼近 B&H 的報酬，可採用。
（未寫進 config.yaml，保留為選項。）

### 結論四（追加）：每次交易金額 — 加碼曲線太淺，加陡到 ratio=2.0（已採納）
針對「每次交易金額」補掃 3 個旋鈕（詳見 [sweep-detail.md](sweep-detail.md) 末段）：
- **`sell_amount`**（每次賣出金額）：現行 10000 已是 Calmar 最佳，丟棄。
- **`buy_base_amount`**（買入基礎額）：純部署量刻度盤，現行 500 是平衡點，丟棄。
- **`buy_tier_ratio`**（加碼曲線陡度）✅ **採納=2.0**：把金字塔由 ~1.5 加陡到 2.0
  （最深 tier 由買 3000 → 8000），中位 **Calmar 1.08→1.10、CAGR +11.9%**，代價 **MaxDD −13.5%→−15.3%**。
  1.75~2.5 為高原（非僥倖）。已寫進 config.yaml（`buy_base_amount: 500` / `buy_tier_ratio: 2.0`）。

**目前 config.yaml 累計採納 2 項**：`sell_rsi_min=75`（免費）＋ `buy_tier_ratio=2.0`（+報酬、+2pp 回撤）。
合計效果：Baseline Calmar 1.01→**1.10**、CAGR +10.1%→**+11.9%**、Calmar 勝率 57%→**62%**、MaxDD −12.9%→−15.3%。

### 跨旋鈕的核心洞察
對 006208 / 00830 這類**倖存者、長期向上、跌了會回來**的大盤型 ETF：
**幾乎所有「加謹慎」的濾網（趨勢濾網、RSI 進場、乖離門檻、止穩確認、保留現金）都會傷害績效**，
因為本策略的 alpha 正是「**無差別買進最深的回檔**」。唯一能**免費**改善風險調整報酬的，是**出場端**
的「賣在 RSI 強勢」。要更高的絕對報酬只能靠**更快部署**（縮短均線 / 放大金額 / 縮短冷卻），代價是更大回撤。

> ⚠️ 倖存者偏誤揭露：以上結論建立在「事後挑選、且都存活上漲」的標的。換到走勢不佳 / 長期下跌的標的，
> 趨勢濾網（`long_ma_above`）很可能反而由負轉正。建議 wave 2 另跑一檔走勢差的對照標的驗證。

## 重現指令

```bash
# 1. 抓資料（一次即可，寫 data/*.csv）
go run ./cmd/fetch_data -months 90

# 2. 三種目標各跑一輪
go run ./cmd/optimize -objective calmar    -passes 3   # 風險調整（穩健，已採納其結果）
go run ./cmd/optimize -objective gates     -passes 3   # 最大化通過關卡數
go run ./cmd/optimize -objective composite -passes 3   # 風險調整×報酬參與
# 報告分別寫到 docs/optimization/sweep-results*.md
```
