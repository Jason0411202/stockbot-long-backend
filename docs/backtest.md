# 回測與績效

本文件說明本專案如何評估策略，以及目前 `00631L + 00830` 的回測結果。數字應以重新執行 `go run ./cmd/eval_csv` 的輸出為準。

## 評估情境

回測採用「期初一次性本金、之後不再從外部投入新資金」（lump-sum 封閉資金池）：

- 期初資金：`initial_cash`，預設 100,000；整個策略只動用這筆錢。
- `monthly_contribution`：定版為 **0**（關閉外部注資）。
- 交易成本目前未計入。

這個設定模擬「投入一筆本金後讓策略自行進出」的使用情境，而不是定期定額或一次性滿倉。
（`monthly_contribution > 0` 仍是支援的選項，會啟用「每月第一個交易日定額注資」情境；下列數字皆為定版 lump-sum。）

## 指標定義

| 指標 | 做法 | 說明 |
| --- | --- | --- |
| MWR / XIRR | 把期初投入（及啟用注資時的每月注資）視為流出，期末清算視為流入，解年化內部報酬率 | 反映每筆錢實際投入多久；封閉資金池時等於 CAGR |
| NAV 最大回撤 | 以單位淨值計算最大回撤（注資視為用當日淨值申購新單位） | 排除注資對權益曲線的灌水 |
| Calmar | `MWR / abs(MaxDD)` | 衡量風險調整後表現 |
| 資金利用率 | 平均持股市值佔總權益比例 | 檢查低回撤是否只是大量抱現金 |

## 對照組

回測同時比較三種組合：

- 策略：使用 `trading.Engine` 執行目前買賣規則。
- B&H：期初就等權買滿追蹤標的，持有到期末。
- Blend：與策略平均曝險相同的市場與現金定權重組合，用來檢查策略是否有真實擇時能力。

三者使用完全相同的起始資金、視窗與交易日歷（啟用注資時也共用同一注資時程）。

## 重現指令

先下載 TWSE CSV 快取：

```bash
go run ./cmd/fetch_data
```

再執行離線評估：

```bash
go run ./cmd/eval_csv
```

若要使用 DB 資料：

```bash
go run ./cmd/evaluate
go run ./cmd/research_run
```

若要對策略旋鈕做暴力網格搜尋（每組都跑全期 + walk-forward + 滾動 IS/OOS，先以四道關卡硬性過濾、再依全期 Calmar 排序，並列出 OOS 保留率與最差折 Calmar 以避免挑到過擬合組合）：

```bash
go run ./cmd/sweep        # 不改 config.yaml,只印排行供人工核可
```

## 目前結果

追蹤標的為 `00631L + 00830`，區間約為 2019-05 到 2026-06。`00631L` 是 2 倍槓桿 ETF，現行參數已針對其波動特性重新調校。

回測起點不是設定值，而是 `CommonIssuanceStart`：「所有追蹤股票都已有資料的最早日」，取各股第一筆資料日的**最大值**，故被**最晚上市**的追蹤股票（`00830`，約 2019-05）綁定——這確保整段期間每檔標的都存在、不在某檔尚未上市的空窗期做決策。`config.yaml` 的 `init_db_back_months` 刻意設成遠超任何資料起點的大值（360 月），讓「資料回補深度」不會成為更緊的天花板；真正的下限永遠由 `00830` 的上市日決定。若要把起點推到 `00631L` 的 2014 上市日，需把 `00830` 移出 `track_stocks`（兩檔聯合回測無法早於 `00830`）。

決策基準為**開盤價**（`decision_price_basis: open`）：以當日開盤價成交、指標只看到前一交易日收盤（見 [strategy.md](strategy.md)）。回測與線上同此基準。下列數字為開盤基準、lump-sum 資金模型重新調參後（`regime_ma_window 85`、`trail_stop_bear 0.08`、`bull_buy_band 0.08`、`cooldown_break_budget 3`、`00631L` 覆寫 `regime_ma_window 60` 與 `trail_reentry_cooldown_days 42`）的結果。

### 全期連續回測

投入本金為 100,000（期初一次性，無外部注資）。區間 2019-05-30 ～ 2026-06-05（約 7.0 年）。

| 指標 | B&H | 策略 |
| --- | --- | --- |
| 期末總權益 | **1,550,914** | 1,321,159 |
| 期末閒置現金 | 約 0 | 445,927 |
| 本金倍數 | **15.51x** | 13.21x |
| MWR | **+47.8%** | +44.4% |
| NAV 最大回撤 | -51.6% | **-24.7%** |
| Calmar | 0.92 | **1.80** |
| 資金利用率 | 100.0% | 68.6% |
| 買/賣次數（停利/了結） | 1/0 | 334/47（35/12） |

策略的報酬略低於槓桿 B&H，但回撤約為其一半，Calmar 約為 B&H 的兩倍。

### Walk-forward 穩健性

設定為 24 個月視窗、每 3 個月步進，共 21 個視窗。

| 指標 | 策略 | B&H | Blend |
| --- | --- | --- | --- |
| 中位 MWR | +35.1% | **+36.6%** | +24.8% |
| 中位 MaxDD | **-19.2%** | -43.4% | - |
| 中位 Calmar | **1.90** | 1.01 | - |
| 資金利用率 | 66.9% | 100.0% | - |
| Calmar 勝率 vs B&H | **95.2%** | - | - |
| 真擇時勝率 vs Blend | **76.2%** | - | - |

目前 scorecard 四道核心關卡（報酬參與、回撤降低、Calmar 勝率、真擇時）皆通過。

### IS/OOS 樣本內外驗證

`cmd/eval_csv` 會執行滾動式 OOS 驗證，用來檢查策略是否只在樣本內好看。現行配置 OOS 中位 Calmar 約 2.43，約為 IS Calmar 1.87 的 **130%**（OOS 比 IS 更佳，非靠壓低 IS 分母），三折 OOS 皆通過 5/5 關卡、最差折 Calmar 1.96，判讀為「穩健」，表示樣本外未退化。

> 註：資金模型由「每月定額注資」改為 lump-sum 後重新以 walk-forward / IS-OOS 掃描調參，發現只需放寬兩個共用旋鈕即可把全期 Calmar 由 1.65 提升到 1.80（`bull_buy_band 0.05→0.08`，讓一次性本金在確認牛市時更快部署，0.08 以上已飽和；`cooldown_break_budget 2→3`，封閉資金池更需把握深跌買點）。`regime_ma_window 85`、`trail_stop_bear 0.08` 等其餘旋鈕重掃後確認仍是最佳點，未因資金模型改變而偏移。

> 暴力搜尋驗證（`cmd/sweep`，8 旋鈕 × 14,580 組合，全期 2019-05～2026-06）：2,136 組通過四道關卡，現行 config 的全期 Calmar 排名第 5。唯一在「Calmar↑ 且 MWR / 回撤 / OOS 最差折皆不輸」嚴格意義上勝過現行的組合，只是把 `bull_buy_band` 0.08→0.11（Calmar 1.80→1.83，噪聲級且與「band 飽和」結論一致），故**不採用**。MWR 更高的組合 OOS 全數退化（保留率 37–46%、最差折 < 0.8）為過擬合，被 OOS 護欄正確擋下。結論：現行參數已是該網格的近最佳解。

per-stock 覆寫採用準則：

- 只接受 IS 表現良好且 OOS 不退化的覆寫。
- `00631L` 使用 `regime_ma_window: 60` 與 `trail_reentry_cooldown_days: 42`。後者為「移動停利出場後暫停逢低買入 ≈42 日」，打斷 2× 槓桿空頭的「停損→又接刀→再停損」whipsaw 循環；lump-sum 重掃確認 cd0（不暫停）全期 Calmar 掉到 1.26、cd30 OOS 保留掉到 83%、cd60 掉關卡，唯 cd42 維持四關全過 + OOS 保留 156%。`regime_ma_window 60` 全期 Calmar 1.80，改回共用 85 則 OOS 保留掉到約 53%（過擬合）。
- `00830` 維持全部共用值。更長 regime 與 reentry cooldown 套到 1× 的 `00830` 在 OOS 退化或拖累報酬，故不採用。

## 注意事項

以上結果未計交易成本，且追蹤標的為事後挑選且長期向上的 ETF。結果適合作為工程與研究參考，不代表未來績效，也不構成投資建議。
