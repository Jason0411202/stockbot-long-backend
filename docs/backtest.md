# 績效

## 問題設定 (problem setting):期初資金 + 每月解鎖新資金

本專案的回測 / 評估情境為**定期定額注資**:

- 期初投入 `initial_cash` (預設 **100,000**)。
- 之後在「**每個日曆月的第一個交易日**」再注入 `monthly_contribution` (預設 **2,500**) 可動用資金 (起始月不注入)。

因為有持續的外部注資,衡量指標也隨之調整 (見 `kernals/walkforward.go`、`kernals/metrics.go`):

| 指標 | 做法 | 為什麼 |
| --- | --- | --- |
| **報酬 = 資金加權年化報酬 (MWR / XIRR)** | 外部現金流 = 期初 + 每月注入 (皆為負)、期末清算 (正),解年化內部報酬率 | 期末權益含大量後期才投入的本金,普通 CAGR(期初→期末) 會嚴重失真;MWR 才反映「投資人實際賺的年化」。`monthly_contribution=0` 時 MWR 恆等於封閉資金池 CAGR (與舊版一致) |
| **回撤 = NAV 單位淨值最大回撤** | 共同基金式單位記帳:注資視為「以當日淨值買單位」,不改變每單位淨值 | 原始權益曲線會被注資灌高、低估回撤;NAV 序列只反映投資績效,其回撤才是真實投資回撤 |
| **B&H 對照組 = 立刻買滿** | 期初買滿,之後每筆解鎖資金一到帳就等權買滿、持有到底 | 對「每月解鎖立刻買」的最自然對照 (永遠盡量滿倉) |
| **同曝險 Blend** | 與策略實際平均曝險 w 相同的「市場 + 現金」定權重組合,收同一份注資 | 防作弊:低持股策略光靠抱現金就能贏 Calmar,策略必須連這個都贏才算真擇時 |

> 兩臂 (策略 / B&H / Blend) 共用**完全相同**的期初資金、注資時程、視窗與交易日歷,故 MWR 可直接比較。

## 重現

先 `go run ./cmd/fetch_data` 抓 TWSE 歷史資料到 `data/*.csv`,再:

- `go run ./cmd/eval_csv` — 離線 CSV、不需 DB:印出「全期連續回測 headline」+「walk-forward 穩健性 scorecard」。
- `go run ./cmd/evaluate` — DB 版、含每視窗明細與完整方法論揭露。
- `go run ./cmd/research_run` — 單一長區間回測 (FinalTotal + 注資合計);本機需要 DB。

> **Task 2 — 提高資金利用率 (已完成)**:原策略在 2023–2026 強多頭裡,+100% 獲利了結不斷把持股換回現金,
> 而牛市買進受冷卻與 10% 現金比例限制、急漲段部署太慢,加上每月持續注資 → 現金越積越多 (期末 $791k、利用率 74%)。

> 結果:利用率 74%→**80%** (wf 79%→**86%**)、閒置現金 $791k→$461k,期末權益與 MWR 反超 B&H,而 NAV 回撤幾乎不變
> (−21.8%→−21.9%)。**取捨**:G2 (中位回撤 ≤60% B&H) 由 60% 放寬到實際 ~68% (−19.6%→−22.1%) —— 提高曝險必然帶來
> 略高回撤,屬刻意取捨;策略仍把 B&H 回撤砍掉約 1/3、Calmar 勝率 95%。實測**收緊熊市移動停利反而因 whipsaw 讓回撤更差**,
> 故維持定版 0.08/0.20;拉高獲利了結門檻也使回撤惡化 (少了 trim 保護),故維持 +100%/賣 33%。

## no-borrow 不變量

可利用資金**僅等於當前持有現金** (起始現金 + 每月注資 + 已實現賣出收入 − 已支出買入成本),不得借錢、不允許透支。
實作上在 `engine.applyBuy` 以 `maxAffordable = floor(cash/price)` 夾取目標股數;夾到 0 則跳過並累計 `SkippedBuys`;
任何時刻 `cash < 0` 立即以不變量違反錯誤中止。
